## Performance Bottlenecks

**Anticipated: Client-Side A* Pathfinding:**
- Problem: The design plan calls for A* pathfinding on a ~1km resolution water grid covering all five Great Lakes. This grid will be large (estimated hundreds of thousands of cells) and pathfinding across long distances (e.g., Duluth to Kingston) could be computationally expensive on the main thread.
- Files: Not yet created (planned as navigation engine component)
- Cause: A* on a large grid with 8-directional movement is O(n log n) in the worst case. The Great Lakes span roughly 1,200km x 600km, yielding a grid of ~720,000 cells.
- Improvement path: Use a Web Worker for pathfinding to avoid blocking the UI. Consider hierarchical pathfinding (HPA*) or pre-computed waypoint graphs for common corridors. Cache computed routes.

**Anticipated: Large GeoJSON Data:**
- Problem: High-detail coastline GeoJSON for all five Great Lakes plus islands will be a large file (potentially several MB).
- Files: Not yet created (planned as map data)
- Cause: High-resolution coastline polygons contain thousands of coordinate pairs per lake.
- Improvement path: Simplify GeoJSON at build time using tools like `mapshaper`. Consider TopoJSON for compression. Lazy-load or code-split the data file so it does not block initial page load.

**Anticipated: SVG Rendering Complexity:**
- Problem: Rendering detailed coastlines, route overlays, grid lines, compass rose, and port markers as SVG could be slow with many DOM elements.
- Files: Not yet created (planned as map renderer)
- Cause: Complex SVG paths with thousands of points create heavy DOM trees.
- Improvement path: Use D3's path simplification. Consider rendering to Canvas for the static map background and overlaying SVG only for interactive elements (ports, route).

## Dependencies at Risk

**jsPDF (v4.2.0):**
- Risk: jsPDF is a large dependency (~300KB+ minified). It is only needed for PDF export, which is a secondary feature.
- Impact: Increases bundle size for all users even if they never export to PDF.
- Migration plan: Lazy-load jsPDF via dynamic import (`import('jspdf')`) so it is only fetched when the user initiates a PDF export. Consider lighter alternatives if PDF quality requirements are simple.

**d3-geo-projection (v4.0.0):**
- Risk: Missing `@types/d3-geo-projection` in devDependencies will cause TypeScript compilation issues.
- Impact: Blocks development of the map projection system.
- Migration plan: Install `@types/d3-geo-projection` or create a local type declaration file.

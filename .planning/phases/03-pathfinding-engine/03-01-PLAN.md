  <name>Task 1: Types, A* engine with BinaryHeap, and path simplification</name>
  <files>lib/pathfinding/types.ts, lib/pathfinding/astar.ts, lib/pathfinding/simplify.ts, lib/pathfinding/__tests__/astar.test.ts, lib/pathfinding/__tests__/simplify.test.ts</files>
  <behavior>
    - A* finds path between two water cells on a small test grid (3x3, 5x5)
    - A* returns null when no path exists (surrounded by land)
    - A* never includes land cells in returned path (verify every cell is water)
    - A* uses diagonal movement -- path from (0,0) to (2,2) on open water grid takes 2 steps not 4
    - Diagonal moves cost sqrt(2), cardinal moves cost 1
    - BinaryHeap extractMin returns lowest priority item
    - BinaryHeap maintains heap property after mixed insert/extract operations
    - simplifyPath with epsilon=0 returns all points
    - simplifyPath removes collinear points (3 points in a line -> 2 points)
    - simplifyPath preserves endpoints and significant turns
    - Performance: A* on real navigation grid (loadGrid()) between distant ports completes in under 2000ms
  </behavior>
  <action>
    Create `lib/pathfinding/types.ts`:
    - `RoutePoint`: `{ lng: number; lat: number }` (geographic coordinate on the route)
    - `PathResult`: `{ points: RoutePoint[]; gridCellCount: number }` (simplified lat/lng path + raw cell count for diagnostics)

    Create `lib/pathfinding/astar.ts`:
    - `BinaryHeap<T>` class: insert(item, priority), extractMin(), size getter. Min-heap using array with bubbleUp/sinkDown.
    - `findPath(grid: NavigationGrid, start: GridCell, end: GridCell): GridCell[] | null`
    - 8-directional movement with DIRECTIONS array: cardinal cost=1, diagonal cost=sqrt(2)
    - Heuristic: octile distance `max(dx,dy) + (sqrt(2)-1) * min(dx,dy)` -- admissible for 8-dir grids, faster than Haversine (per research, this is a refinement of the Haversine decision -- both produce identical optimal paths but octile is standard for grid A*)
    - Use flat Map keyed on `row * grid.width + col` for gScore/cameFrom lookups (faster than string keys)
    - Guard: `isWater(grid, col, row)` on every neighbor expansion
    - Reconstruct path by walking cameFrom from end to start, then reverse

    Create `lib/pathfinding/simplify.ts`:
    - `simplifyPath(points: [number, number][], epsilon: number): [number, number][]`
    - Douglas-Peucker algorithm: recursively find point with max perpendicular distance from line between endpoints, split if > epsilon
    - `perpendicularDistance` helper function
    - Return endpoints only if no point exceeds epsilon

    Write tests FIRST (RED), then implement (GREEN):
    - `lib/pathfinding/__tests__/astar.test.ts`: Test findPath on small grids (create inline NavigationGrid objects), test BinaryHeap operations, test land avoidance, test diagonal movement, test performance on real grid
    - `lib/pathfinding/__tests__/simplify.test.ts`: Test simplifyPath with collinear points, L-shaped paths, epsilon=0, single/two points
  </action>

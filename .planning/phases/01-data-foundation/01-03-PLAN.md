  <name>Task 1: Create coastline and waterway validation tests</name>
  <files>lib/geo/__tests__/coastlines.test.ts, lib/grid/__tests__/waterways.test.ts, lib/grid/__tests__/performance.test.ts</files>
  <action>
1. Create `lib/geo/__tests__/coastlines.test.ts` (DATA-01 validation):

   Tests:
   - `loadCoastlines()` returns a FeatureCollection with features
   - Feature count is >= 5 (at least one per Great Lake)
   - TopoJSON source file size is under 500KB (read `lib/geo/great-lakes.topo.json` with `fs.statSync`)
   - The coastline data, when converted to GeoJSON, contains features that cover the full Great Lakes bbox extent (check that features span from ~-92 to ~-76 longitude and ~41 to ~49 latitude)
   - Spot-check: geoContains test for known water points (center of each lake):
     - Lake Superior: [-87.5, 47.5]
     - Lake Michigan: [-86.5, 43.5]
     - Lake Huron: [-82.0, 44.5]
     - Lake Erie: [-81.0, 42.2]
     - Lake Ontario: [-77.5, 43.5]
   - Spot-check for known land points (should NOT be contained):
     - Downtown Detroit: [-83.04, 42.33]
     - Midland Ontario: [-79.88, 44.75]

   Use `d3-geo` `geoContains` for point-in-polygon tests.

2. Create `lib/grid/__tests__/waterways.test.ts` (DATA-02 validation):

   Tests:
   - Grid loads successfully with valid structure (has width, height, bbox, cellSize, data)
   - Grid dimensions match expected (~1700 x ~850 at 0.01 degree resolution, or proportional if different cellSize)
   - Water cell percentage is between 15-50% (sanity check -- Great Lakes are about 20-30% of the bbox area)
   - Each of the 5 connecting waterways has water cells at known coordinates:

     a. **Straits of Mackinac** -- test 3 points along the strait:
        - [-84.9, 45.78], [-84.7, 45.80], [-84.6, 45.82]

     b. **St. Marys River** -- test 3 points:
        - [-84.35, 46.48], [-84.35, 46.43], [-84.35, 46.38]

     c. **Detroit River / Lake St. Clair** -- test 3 points:
        - [-82.42, 42.95] (St. Clair River)
        - [-82.7, 42.45] (Lake St. Clair area)
        - [-83.1, 42.3] (Detroit River)

     d. **Welland Canal** -- test 3 points along the corridor:
        - [-79.22, 42.90], [-79.22, 43.05], [-79.22, 43.15]

     e. **Upper St. Lawrence** -- test 3 points:
        - [-76.3, 44.2], [-76.0, 44.3], [-75.7, 44.4]

   For each waterway, use `toCell()` + `isWater()` to verify. If a point fails, the test message should clearly name the waterway and coordinates.

   - **Connectivity test**: For each waterway, verify that adjacent cells form a connected path (not just isolated water cells). Pick two points on opposite sides of the waterway and verify a chain of water cells connects them using BFS within a small region (20x20 cell neighborhood).

3. Create `lib/grid/__tests__/performance.test.ts` (DATA-02 performance):

   Tests:
   - Grid data loads (import or fetch) in under 1 second. Time the `loadGrid()` call.
   - `toCell()` executes in under 1ms for a single lookup (run 1000 lookups, total should be < 100ms).
   - `isWater()` executes in under 1ms for a single lookup (run 1000 lookups, total should be < 100ms).
  </action>

  <done>Coastline tests confirm all 5 Great Lakes present with correct containment. Waterway tests confirm all 5 connecting waterways have navigable water cells. Performance tests confirm grid operations complete within time limits.</done>

   This is the critical integration test that verifies ports are actually reachable on the navigation grid.

   Tests:
   - For EVERY port in the database, verify that `toCell(grid, port.lng, port.lat)` produces a valid cell within grid bounds
   - For EVERY port, verify that EITHER:
     a. The port's cell is water (`isWater` returns true), OR
     b. There is a water cell within 5 cells (~5km) found by `findNearestWaterCell`
   - Count and report: how many ports are directly on water vs. need snapping
   - FAIL if more than 10% of ports need snapping (indicates bad coordinate data)
   - FAIL if ANY port has no water cell within 10 cells (~10km) radius
   - Group ports by lake and verify each lake has ports that map to valid water cells

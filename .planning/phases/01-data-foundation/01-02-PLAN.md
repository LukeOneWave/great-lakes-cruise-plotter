  <name>Task 1: Create waterway corridors and geo processing script</name>
  <files>lib/geo/waterway-corridors.json, scripts/prepare-geo.ts, package.json</files>
  <action>
1. Create `lib/geo/waterway-corridors.json` -- a GeoJSON FeatureCollection defining wider-than-reality corridor polygons for narrow waterways. These corridors ensure the navigation grid marks these areas as water even though they are narrower than grid resolution. Define corridors for ALL 5 connecting waterways:

   a. **St. Marys River** (connects Superior to Huron): Corridor from approximately [-84.55, 46.25] to [-84.25, 46.55]. The river runs through Sault Ste Marie. Make corridor ~5km wide to capture the navigable channel.

   b. **Straits of Mackinac** (connects Michigan to Huron): Corridor from approximately [-85.0, 45.72] to [-84.6, 45.85]. This is wider naturally (~6km) but ensure full coverage.

   c. **St. Clair River + Lake St. Clair + Detroit River** (connects Huron to Erie): This is a system. St. Clair River runs from Port Huron/Sarnia [-82.45, 42.95] to [-82.55, 43.00] down to Lake St. Clair. Detroit River from [-83.15, 42.25] to [-82.95, 42.35]. Lake St. Clair itself should be in Natural Earth data, but add a corridor linking the river sections.

   d. **Welland Canal** (connects Erie to Ontario): Corridor from approximately [-79.25, 42.87] to [-79.20, 43.22]. This is the narrowest (~300m real width) and MUST have a corridor. Make it ~5km wide in the GeoJSON.

   e. **Upper St. Lawrence River** (connects Ontario eastward): Corridor from approximately [-76.5, 44.1] to [-75.5, 44.7]. Runs from Kingston area eastward.

   Each feature should have properties: `{ "name": "...", "type": "canal|river|strait|channel" }`.

2. Add npm scripts to `package.json`:
   ```json
   "prepare-geo": "npx tsx scripts/prepare-geo.ts",
   "generate-grid": "npx tsx scripts/generate-grid.ts",
   "prepare-data": "npm run prepare-geo && npm run generate-grid"
   ```
   Also install tsx as a dev dependency: `npm install -D tsx`

3. Create `scripts/prepare-geo.ts`:
   This build script downloads Natural Earth 10m data and produces the TopoJSON file.

   Pipeline steps:
   a. Download Natural Earth shapefiles if not cached locally:
      - `ne_10m_lakes.shp` (+ .dbf, .shx, .prj) from `https://naciscdn.org/naturalearth/10m/physical/ne_10m_lakes.zip`
      - `ne_10m_lakes_north_america.shp` from `https://naciscdn.org/naturalearth/10m/physical/ne_10m_lakes_north_america.zip`
      Download to `data/` directory (create if not exists). Use Node.js `fetch` + `unzip` (or extract manually). Add `data/` to `.gitignore`.

   b. Read shapefiles using the `shapefile` npm package (`read` function).

   c. Filter features to Great Lakes region:
      - Filter by name property matching any of: 'Lake Superior', 'Lake Michigan', 'Lake Huron', 'Lake Erie', 'Lake Ontario', 'Lake St. Clair'
      - Also include any feature from ne_10m_lakes_north_america that falls within the bounding box [-92.5, 41.0, -75.5, 49.5] (captures smaller lakes and bays)
      - Log all feature names found for debugging

   d. Merge waterway corridor polygons from `lib/geo/waterway-corridors.json` into the feature collection. These corridors become additional water polygons.

   e. Convert to TopoJSON using `topojson-server` (`topology` function).

   f. Simplify using `topojson-simplify` (`presimplify`, `simplify`, `quantile`). Start with `quantile(topo, 0.02)` to keep 98% of detail. If file > 500KB, increase to 0.03 or 0.04 iteratively.

   g. Quantize with `topojson-client` `quantize` at 1e5.

   h. Write to `lib/geo/great-lakes.topo.json`.

   i. Log: file size in KB, number of features, number of arcs.

   j. Validate: Assert file < 500KB. Assert all 5 lake names present in output. Assert feature count > 5.

   IMPORTANT: The `shapefile` package's `read` function returns a GeoJSON FeatureCollection. Use `import * as shapefile from 'shapefile'` or `const shapefile = require('shapefile')` depending on module resolution. Check the API: `shapefile.read(filename)` returns a Promise of GeoJSON FeatureCollection.

   For downloading and unzipping, use Node.js built-in `fetch` (available in Node 18+) and a simple unzip approach. Consider using `child_process.execSync` to call `curl` and `unzip` system commands for simplicity, since this is a build script.
  </action>

<task type="auto">
  <name>Task 2: Create grid generation script and runtime loaders</name>
  <files>scripts/generate-grid.ts, lib/geo/load-geo.ts, lib/grid/grid.ts</files>
  <action>
1. Create `scripts/generate-grid.ts`:
   This build script reads the TopoJSON output from prepare-geo and rasterizes it into a navigation grid.

   Steps:
   a. Read `lib/geo/great-lakes.topo.json` and convert to GeoJSON using `topojson-client` `feature()`.

   b. Define grid config:
      - bbox: [-92.5, 41.0, -75.5, 49.5] (Great Lakes bounding box)
      - cellSize: 0.01 degrees (~1km at this latitude)
      - This gives approximately 1700 x 850 = ~1.4M cells

   c. For each cell center, test if it falls inside any water polygon using `d3-geo` `geoContains()`. Mark as 1 (water) or 0 (land).

   d. Store as `NavigationGrid` JSON matching the interface from `lib/grid/types.ts`:
      ```json
      { "width": N, "height": N, "bbox": [...], "cellSize": 0.01, "data": [0,1,0,...] }
      ```

   e. Write to `lib/grid/navigation-grid.json`.

   f. Validate the grid:
      - Log total cells, water cells, land cells, water percentage
      - Check specific waterway points to verify they are marked as water:
        - Straits of Mackinac: [-84.8, 45.8] should be water
        - St. Marys River: [-84.35, 46.5] should be water
        - Detroit River area: [-83.1, 42.3] should be water
        - Welland Canal: [-79.22, 43.0] should be water
        - Upper St. Lawrence: [-76.0, 44.3] should be water
      - If any waterway check fails, log an ERROR (do not silently succeed)

   g. Log file size. The JSON with the flat data array will be large (~3-5MB uncompressed). This is acceptable because it compresses well with gzip (~200KB) and Next.js serves compressed assets. If the JSON is too large for a static import, consider storing the data array as a compact format (e.g., run-length encoding or base64-encoded Uint8Array).

   PERFORMANCE NOTE: Rasterizing 1.4M cells with geoContains is slow (potentially 10-30 minutes). To speed up:
   - Skip cells that are clearly outside the lakes' bounding boxes
   - Process in batches and log progress every 10%
   - Consider a coarser initial pass (0.05 degree) to identify water regions, then refine

   If performance is unacceptable, reduce resolution to 0.02 degrees (~2km), giving 850x425 = ~360K cells. This is still adequate for pathfinding.

2. Create `lib/geo/load-geo.ts`:
   Runtime module that loads TopoJSON and converts to GeoJSON.

   ```typescript
   import { feature } from "topojson-client";
   import type { Topology } from "topojson-specification";
   import type { FeatureCollection } from "geojson";
   import topoData from "./great-lakes.topo.json";

   export function loadCoastlines(): FeatureCollection {
     const topo = topoData as unknown as Topology;
     const objectKey = Object.keys(topo.objects)[0];
     return feature(topo, topo.objects[objectKey]) as unknown as FeatureCollection;
   }
   ```

   If static JSON import fails due to file size, use dynamic `fetch('/great-lakes.topo.json')` instead and move the file to `public/`.

3. Create `lib/grid/grid.ts`:
   Runtime module that loads the navigation grid and provides cell lookup.

   - `loadGrid(): NavigationGrid` -- imports and returns the grid data
   - Re-export `toCell` and `isWater` from `lib/grid/types.ts` (they are already implemented there per Plan 01)
   - Add `findNearestWaterCell(grid: NavigationGrid, lng: number, lat: number, maxRadius?: number): GridCell | null` -- for snapping ports that fall on land to the nearest water cell. Search in expanding rings around the target cell up to maxRadius (default 10 cells).

   For the grid JSON import, if it is too large for a TypeScript static import, use a dynamic approach:
   - Move `navigation-grid.json` to `public/data/navigation-grid.json`
   - `loadGrid()` becomes async and uses `fetch('/data/navigation-grid.json')`
